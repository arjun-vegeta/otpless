# Check SNA status

> Check the status of a Silent Network Authentication (SNA) transaction. Use only with requestId from Initiate SNA.

<Accordion title="When to call the Status API">
  Call this endpoint with the **requestId** returned from [Initiate SNA](/api-reference/endpoint/sign-in/sna/initiate-sna) after the user has opened the verification link on their device. Poll until you receive a terminal status: `SUCCESS` or `FAILED`. The final authentication decision should always come from this API, not from the redirect alone.
</Accordion>

<Accordion title="Status values">
  | Status    | Type   | Description                                                     |
  | --------- | ------ | --------------------------------------------------------------- |
  | `INIT`    | string | Session created successfully. Awaiting device to open the link. |
  | `PENDING` | string | Link opened on device. Telco handshake in progress.             |
  | `SUCCESS` | string | Phone number verified successfully.                             |
  | `FAILED`  | string | Verification failed. See error object for details.              |
</Accordion>

<Accordion title="Polling guidance">
  Start polling after the link is opened. Poll every 2–3 seconds. Stop when you receive `SUCCESS` or `FAILED`, or after your client-side timeout (e.g. 30–60 seconds). Do not poll aggressively.
</Accordion>
